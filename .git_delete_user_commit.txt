fix(delete-user): correct delete path with safety check

### Issue Fixed
Added missing DELETE route and destroy action for user account deletion with proper authorization.

### Changes Made

1. config/routes.rb
   - Added DELETE route mapped to users/circuitverse#destroy
   - Enables DELETE /users/:id requests to be properly routed

2. app/controllers/users/circuitverse_controller.rb
   - Added :destroy to authenticate_user! filter
   - Added authorize_user_deletion before_action filter
   - Implemented destroy action with:
     * User account deletion via destroy!
     * Redirect to root with success notice
     * Error handling with logging
   - Added authorize_user_deletion private method:
     * Safety check: Only users can delete their own account
     * Prevents unauthorized deletion attempts
     * Redirects with alert on unauthorized access

### Safety Checks
Authentication required (authenticate_user!)
Authorization check (users can only delete their own account)
Error handling with proper logging
Graceful failure with user-friendly messages

### How It Works
1. User clicks delete account button in profile edit page
2. Modal confirmation dialog appears
3. JavaScript sets delete_path to /users/{user_id}
4. User confirms deletion
5. DELETE request sent to /users/{user_id}
6. Route directs to Users::CircuitverseController#destroy
7. Authorization check verifies user is deleting their own account
8. Account deleted
9. Redirected to home page with success message
